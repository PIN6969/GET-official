# Official Site Deployment Guide

This project must be deployed using the build output in `dist`.

## 1) Build

Run in project root:

```bash
npm ci
npm run build
```

## 2) Files to Deploy

Only deploy the contents inside `dist/`.

- Correct: upload all files/folders under `dist/`
- Incorrect: upload project source files like `src/`, `index.html` (root), `contact.html` (root)

## 3) Server Path Notes

- `vite.config.js` uses `base: './'`, so the site supports subfolder deployment.
- Keep folder structure exactly as generated by Vite.

## 4) Quick Verification After Deploy

Open site and check:

1. Home page has full styles (not plain text layout).
2. Browser Network tab shows `assets/main-*.css` and `assets/main-*.js` loaded with status 200.
3. `contact.html` opens correctly and uses same styles.

## 5) GitHub Pages (Optional)

If using GitHub Pages, prefer GitHub Actions deployment with `.github/workflows/deploy-pages.yml`.
